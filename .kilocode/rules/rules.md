# Правила для LLM


## Общие правила
- НЕ создавать новые папки без запроса
- НЕ изменять файлы вне описанного контекста
- Предпочитать минимальные изменения
- Всегда отвечай на русском языке


## Backend
- Соблюдать PEP8
- Использовать type hints
- Не добавлять сложные абстракции без необходимости


## Frontend
- Строго использовать TypeScript
- Не добавлять UI-библиотеки без явного запроса


## Безопасность
- Никогда не изменять переменные окружения
- Никогда не удалять файлы без подтверждения


## Процесс работы
- Если есть сомнения — задавать уточняющий вопрос
- Кратко объяснять логику при генерации кода
